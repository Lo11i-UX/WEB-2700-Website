Introduction: Why visual design and structure

Good visual design is important because it creates a better user experience. When a website looks clean and easy to use visitors are more likely to stay longer using your website. And those with disabilities will be able to use your website easily.

If a website has a good structure, it allows ease of navigation for users. Accessibility for those using screen readers. Also allows good responsiveness for mobile designs.

Visual Design:

Colors:

Primary Color: Blue (#136172).

At first, I was using bootstrap bg-info and added bg-opacity-50 because I did not like how bright the bg-info was. However, checking the Contrast Check it failled the WCAG AAA check. So I manually selected it to (#136172).

Secondary Colors:

  • Bg-info (#0dcaf0):. I used the bg-info form bootstrap as the color for the naviagation.
  • Aqua Blue (#00D0FF): for the footer.
  • bg-dark (#212529): Used for the page body background for a nice dark mode experiance.
  • table-primary (#212529): Used for the tables.
  • bg-primary bg-opacity-75 (#4A92FE): Used for the Accordions.
  • bg-info-subtle (#CFF4FC): Used for the contact form backgrounds.

The way I came accross these colord where basing it on the bootstrap premade colors. And the color --primary was the best suited for the darker background. I used bg-opacity if I wanted variations on the primary blue color.

Typography:

The fonts I used where: "system-ui,-apple-system,"Segoe UI",Roboto,"Helvetica Neue","Noto Sans","Liberation Sans",Arial,sans-serif,"Apple Color Emoji","Segoe UI Emoji","Segoe UI Symbol","Noto Color Emoji". this choice is because i used bootstrap and it imported "--bs-body-font-family".

I used bootstrap defaults for the sizing, weights and colors for almost all of the website. Entrusting that the bootstrap developers made everything appropriately.

Layout/Navigation

I caused the navigation to be located at the top center of the page, causing a eye motion of top down. If the screen size gets too small, there is a hamburger icon displayed instead.

Content

I based the content based on past assignemnts of this class and the requirements for the final project. Such as Home, Plain Language, Site Design, About Me, Contact Me

Resized without compromising readability?

Yes, because it was build upon bootstrap, causing the page struture to expand and condense naturally.

Bootstrap?

YES! Bootstrap was heavly used in this website for it's easy of use and bulit in accessibility capabilities.

Structural Design

Many things where implmenented to insurance good accessibility design from a coding standpoint.

  • Used ARIA
  • Used semantic elements as much as possibile, such as h1, h2, p, etc
  • Descripitive link text: All the links have descripitive text such as the reference section in the home page.
  • Focus styles: Relied upon bootstrap to have focus capabilities built in. But for the header navigation I added a class that will highlight box will be on the selected item.
  • Skip navigation link: I added a skip navigation button for screen readers to say and users to select with there keyboard.
  • Responsive design: Since almost all the website is made with bootstrap, it has response layout built into it.
  • Alt-Text is added to all the images and figures in the website.
  • Tested with windows screen reader. Checked for accessibility scores with lighthouse and axe Devtools. Checked colors with contrast checker tool.
